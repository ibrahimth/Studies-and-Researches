Que outros insights você é capaz de obter com base nos dados fornecidos?


As medidas estatísticas solicitadas na questão 1 (mínimo, máximo, média, mediana e moda) avaliam a tendência dos dados.
Outras cálculos que podem ser feitos e que possuem grande relevância sobre a qualidade dos dados são a variância e o 
desvio padrão. Com a variância e o desvio padrão em mãos, é possível medir outliers e avaliar a necessidade de normalização 
dos dados para melhorar as análises preditivas.

Aplicar a moda a outras informações permite gerar agrupamentos logísticos mais consistentes para avaliar por exemplo
o se o "tempo total de ligação" de um atendimento foi bom ou ruim. Não necessariamente um tempo total de ligação longe
da média é ruim. Cruzar a moda do "tempo total de ligação" com a coluna de "aceite de oferta" permite checar por
exemplo o quanto ligações com tempo de ligação alto são efetivas. Não é uma análise totalmente certa, mas aumenta
a confiança em outras deduções. É possível realizar o mesmo tipo de análise com outras informações como profissão e 
nível educacional por exemplo.

Outra avaliação possível seria checar a qualidade da base de contatos através de um mapa de frequência. Em cenários
como esse é comum que a empresa que oferece o produto (no caso o seguro de automóveis) possua um perfil de cliente
com maior potencial de venda bem definido. Utilizando as informações desse perfil, é possível criar um mapa de frequência
e verificar o quanto a base de dados está próxima do perfil de cliente ideal levantado. Obviamente deve ser considerado 
que novos nichos de clientes podem ser encontrados para o produto e que essa descoberta somente pode ser feita explorando 
novas informações, mas considerando que quanto maior o volume de dados, maior é a tendência a média, se o perfil de 
potencial cliente já está bem definido, dificilmente uma base de dados de contato muito longe desse perfil ideal trará 
resultados efetivos em relação a outras bases que tem alta aderência ao perfil de cliente da empresa.

Uma última consideração que gostaria de ressaltar é que é possível utilizar a mesma base de dados para obter outras
previsões diferentes da direcionada nesse contexto (no caso "aceite de oferta"). Olhando os dados por outro ângulo, pode ser 
interessante em outros cenários avaliar por exemplo informações como a quantidade de ligações necessárias para que um cliente
aceite a oferta, melhores horários para ligação, melhor meio de comunicação e a quantidade de dias ideal entre os contatos.
Pode ser necessário agregar outros dados a predição (e neste caso eu acredito que será), mas a utilização de modelos
neurais regressores e probabilísticos permite estimar informações como essas e aumentar os resultados obtidos nos processos 
empresariais como um todo. Existe ainda a possiblidade de trabalhar com modelos treinados por reforço para criar agentes 
inteligentes somente utilizando métricas de alto nível como fonte de aprendizado. Neste caso, a necessidade
de uma base de dados é mínima e na maioria dos casos, somente com conhecimento sobre o negócio, é possível criar um 
ambiente capaz de fornecer dados para o treinamento de agentes inteligentes que conseguem aprender gerando informações 
para o seu próprio treinamento por exploração ("tentativa e erro").
